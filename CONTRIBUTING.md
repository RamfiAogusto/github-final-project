# Guía de Contribución

¡Gracias por tu interés en contribuir a nuestro proyecto! Valoramos mucho todas las contribuciones.

## Cómo Contribuir

### Reportar Bugs

Si encuentras un error, por favor crea un issue con:
- Una descripción clara del problema
- Pasos para reproducir el error
- Información sobre tu ambiente (SO, versión, etc.)
- Capturas de pantalla si es relevante

### Sugerir Nuevas Características

Para sugerir nuevas características:
- Abre un issue con la etiqueta "enhancement"
- Describe claramente la funcionalidad que propones
- Explica por qué sería útil para el proyecto
- Incluye ejemplos si es posible

### Enviar Pull Requests

1. **Fork el repositorio**
2. **Crea una rama para tu característica**
   ```bash
   git checkout -b feature/nueva-caracteristica
   ```
3. **Haz tus cambios**
4. **Añade o actualiza tests si es necesario**
5. **Asegúrate de que el código sigue las convenciones del estilo**
6. **Commit tus cambios**
   ```bash
   git commit -m "Agregar nueva característica: descripción"
   ```
7. **Push a tu rama**
   ```bash
   git push origin feature/nueva-caracteristica
   ```
8. **Abre un Pull Request**

### Convenciones de Código

- Usa nombres descriptivos para variables y funciones
- Comenta el código cuando sea necesario
- Sigue las mejores prácticas de Bash para scripts
- Mantén las líneas de código bajo 80 caracteres cuando sea posible

### Mensajes de Commit

Usa mensajes de commit claros y descriptivos:
- `feat: agregar nueva funcionalidad`
- `fix: corregir error en cálculo`
- `docs: actualizar documentación`
- `style: mejorar formato del código`
- `refactor: reestructurar código`
- `test: agregar pruebas`

## Configuración del Desarrollo

### Prerrequisitos

- Git
- Bash (para sistemas Unix/Linux/macOS)
- Editor de texto de tu preferencia

### Configuración Local

1. Fork y clona el repositorio
2. Crea una rama para tu trabajo
3. Realiza tus cambios
4. Prueba el script localmente
5. Envía el pull request
