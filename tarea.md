Antes de enviar su trabajo y revisar el de sus compañeros, asegúrese de haber completado la parte del proyecto correspondiente a los laboratorios prácticos. 

Hay un total de 20 puntos por 10 tareas en este proyecto final repartidas en dos laboratorios prácticos. Su tarea final será calificada por sus compañeros que también estén completando esta tarea en la misma sesión. Deberá enviar los siguientes elementos para que los revisen sus compañeros, y el formato exacto se especifica en cada tarea.

Un envío de URL para 4 tareas

Un envío de texto para 5 tareas

Un envío de captura de pantalla para 1 tarea

Éstas enlazarán con su proyecto para que sus compañeros puedan revisarlo.

Criterios de calificación - 20 puntos en total

Los criterios de calificación de la tarea final para las dos partes son los siguientes:

Parte 1 Interfaz de usuario de GitHub (12 puntos)

Tarea 1: Envíe la URL del repositorio. (2 puntos)

0 puntos: No se envía la URL.

1 pto: Se envía una URL válida pero la URL está vacía.

2 ptos: URL válida enviada y la URL tiene todos los archivos como se esperaba.

Tarea 2: El archivo de licencia Apache 2.0 llamado LICENSE está presente en el repositorio. (2 ptos)

0 ptos: El repositorio no tiene un fichero de licencia.

1 pto: El repositorio tiene un fichero de licencia, que es diferente de la especificación.

2 ptos: El repositorio tiene un fichero de licencia Apache 2.0.

Tarea 3: El archivo README.md está presente en el repositorio. (2 ptos)

0 ptos: El repositorio no tiene archivo README.

1 pto: El repositorio tiene un archivo README, que es diferente de la especificación o no tiene contenido.

2 ptos: El repositorio tiene el archivo README.md según la especificación.

Tarea 4: El archivo CODE_OF_CONDUCT está presente en el repositorio. (2 ptos)

0 ptos: El repositorio no tiene un fichero CODE_OF_CONDUCT.md.

1 pto: El repositorio tiene un archivo CODE_OF_CONDUCT, que es diferente de la especificación o no tiene contenido.

2 ptos: El repositorio tiene un fichero CODE_OF_CONDUCT conforme a la especificación.

Tarea 5: El archivo CONTRIBUTING.md está presente en el repositorio. (2 ptos)

0 ptos: El repositorio no tiene un fichero CONTRIBUTING.md.

1 pto: El repositorio tiene un fichero CONTRIBUTING, que es diferente de la especificación o no tiene contenido.

2 ptos: El repositorio tiene un fichero CONTRIBUTING según la especificación.

Tarea 6: El fichero simple-interest.sh está presente en el repositorio. (2 puntos)

0 ptos: El repositorio no tiene un fichero simple-interest.sh.

1 pto: El repositorio tiene un fichero simple-interest.sh, que es diferente de la especificación o no tiene contenido.

2 ptos: El repositorio tiene un archivo simple-interest.sh según la especificación.

Parte 2 Git CLI (8 puntos)

Tarea 1: Envíe la URL del repositorio bifurcado. (2 ptos)

0 ptos: La URL no se envía o el enlace no funciona.

1 pto: La URL es un enlace válido, pero el contenido no está bifurcado correctamente.

2 ptos: La URL es un enlace válido y es la bifurcación del 
repositorio
 dado con todos los contenidos.

Tarea 2: Envíe la captura de pantalla de la primera bifurcación. (2 ptos)

0 ptos: La imagen no se envía.

1 pto: La imagen enviada muestra que la rama bug-fix-typo está fusionada en main pero los detalles no son del todo correctos.

2 ptos: La imagen muestra que la rama actual es main y los cambios en el archivo README.md en bug-fix-typo han sido fusionados a main.

Tarea 3: Envíe la URL del pull request. (2 ptos)

0 ptos: No se envía la URL.

1 pto: La URL funciona pero no apunta al pull request.

2 ptos: La URL funciona y refleja el pull requested que ha sido aceptado.

Tarea 4: Envíe la URL de la página Branches que muestra los nombres de las ramas y su estado en su repositorio. (2 ptos)

0 ptos: La URL no se envía.

1 pto: La URL sólo muestra el número de ramas como 3 viz (main, bug-fix-typo y bug-fix-revert), pero los detalles no son evidentes.

2 ptos: La URL muestra los nombres de las ramas y el estado.
